# Router

The Router class gives you the ability to register routes and a corresponding handler for that route.

# constructor

When creating a Route you have already seen that we pass a string into the constructor that is actually not required as it will default to an empty string. That string represents a prefix for your router. For example in the case you want to create a users router you can prefix your routes with /users and then you don't need to manually write /users in front of every route declaration.

import {Router} from "router";

const router = new Router("/users");

router.get("/:id", async () => {}); // will result in /users/:id

# HTTP methods

# use(handler: (context: Context) => void)

The use method allows for applying features to single route definitions but also to the whole router itself. Here the order of method calls matters. As soon as one route handler has been added features will always be added to the last added route handler. If no route handler has been added the feature will be router global.

const router = new Router();

function IWillRunOnEveryRoute() {}
function IWillRunOnlyAtThisRoute() {}

router.use(IWillRunOnEveryRoute);

router.get("/", () => {}).use(IWillRunOnlyAtThisRoute);
