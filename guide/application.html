# Application

In the getting started section you have seen that the Application is were everything comes together and you have seen methods like use and listen but there is much more to be discovered. Depending on what parameters are given to the constructor a few more things can happen. Based on the parameters listen will start a http or https server. If either key or cert is missing the app will fall back to creating a http server.

# constructor

When creating an instance of Application the constructor takes an object where the port field is required. key and cert are optional and are used to create a https server.

import {Application} from "pine";

const app = new Application({
  port: 80,
  key?,
  cert?,
});

# listen

listen will start the server with the given configuration.

# use(feature: Router|Router[])

There is not much new to learn regarding the use method appart from the fact that it can also receive an array of Router instances.









 
 
 
 

import {Application} from "pine";

const app = new Application({
  port: 80,
  key?,
  cert?,
});

app.use([
  router,
  anotherRouter
]);

# dispose

dispose will get rid of the running http/s server. This can be helpful when testing your application with for example jest.

# server

server is a getter method that will return the running http/s server. This will come in handy when integrating socket.io.

import {Server} from "socket.io";
import {Application} from "pine";

const app = new Application({
  port: 80,
});

const io = new Server(app.server());

app.listen();

# Events

Lastly the Application class also provides an event system which can be used with the on and dispatch methods.

import {Application} from "pine";

const app = new Application();

app.on("ready", () => {
  console.log("Server up and running...");
});

There are a couple of events that will be dispatched automatically so you only have to listen to them if you want to. For example the ready event will be dispatched when the server is listening and shutdown when the server is disposed or the programm is closed using a keyboard interrupt like ctrl+c.
